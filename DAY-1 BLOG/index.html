1) Write a blog on Difference between HTTP1.1 vs HTTP2

Introduction on HTTP

HTTP is a request-response protocol. This means that the browser sends a request to the server, and the server sends a response back

HTTP is a client-server protocol, which means that it involves communication between a client such as a web browser or mobile app

HTTP/1.1 vs HTTP/2

Feature HTTP/1.1 HTTP/2
Threaded Single-threaded Multi-threaded
Protocol Text-based Binary
Performance slow faster

HTTP/1.1 is the original version of HTTP. It's a single-threaded protocol, which means that it can only send one request at a time. This can lead to performance problems, especially on slow connections

HTTP/2 is a newer version of HTTP that addresses the performance problems of HTTP/1.1. It's a multiplexed protocol, which means that it can send multiple requests and responses over a single connection. This significantly improves performance, especially on slow connections.

3) Read about IP address, port, HTTP methods, MAC address

IP Address:

Internet Protocol Address is a unique numerical (Logical Address) label assigned to each device connected to a computer network.

It act as an identifier for both the source and destination of communicating.

IP addresses are divided into two:

• Public IP addresses: Used to identify on Internet.

• Private IP addresses: Used to identify on Local Network.

Port:

A port is a logical endpoint on a device.

used to listen or to send data.

Ports are identified by number.

HTTP Methods:

HTTP methods define the type of action to be performed based on a URL. The common HTTP methods are:

  • GET: Retrieves data from a server.
  • POST: Sends data to a server to create a new resource.
  • PUT: Updates an existing resource on a server.
  • DELETE: Deletes a specified resource on a server.
